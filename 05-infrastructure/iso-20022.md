# ISO 20022 — Global Financial Messaging Standard

## 🧭 What is it
A universal financial messaging standard for electronic data interchange between financial institutions. Replacing legacy formats (SWIFT MT, ISO 8583, EDIFACT) with rich, structured XML/JSON messages.

---

## 🧱 Core Components

- **Message Repository** — Catalog of all standardized message definitions
- **Business Domains** — Payments, Securities, Trade Finance, FX, Cards
- **Message Naming Convention** — `<domain>.<function>.<variant>.<version>`
  - Example: `pacs.008.001.10` = Payment Clearing & Settlement, Financial Institution Credit Transfer, variant 1, version 10
- **Data Model** — Standardized data elements (LEI, IBAN, BIC, etc.)
- **Governance** — ISO Technical Committee 68 (TC68), Registration Authority (RA)

---

## 🔄 How ISO 20022 Works

```
Financial Institution A
  │ Creates ISO 20022 XML message
  │ (e.g., pacs.008 = credit transfer)
  ▼
SWIFT Network / Local ACH / RTGS
  │ Validates message schema
  │ Routes based on structured data
  ▼
Financial Institution B
  │ Receives structured message
  │ Extracts: amount, IBAN, BIC, purpose, LEI
  ▼
Straight-Through Processing (STP) — no manual intervention
```

## ⚖️ ISO 20022 vs Legacy Formats

| Feature | ISO 20022 | SWIFT MT | ISO 8583 |
|---------|-----------|---------|---------|
| Format | XML / JSON | Fixed-length text | Binary fixed-length |
| Data richness | Rich (unlimited fields) | Limited | Card-specific |
| Extensibility | High | Low | Low |
| STP rate | Very high | Medium | Card-specific |
| Use case | All payments | Cross-border messaging | Card authorization |
| Remittance data | Full structured | 140 char limit | Not applicable |
| LEI support | Native | Limited | No |

---

## ⚙️ Benefits of ISO 20022

```
Enhanced data = Better automation:

  1. Full remittance information → automated reconciliation
  2. Structured address → faster OFAC/AML screening
  3. LEI (Legal Entity Identifier) → entity verification
  4. Purpose code → routing and reporting
  5. Rich creditor reference → straight-through processing
  6. Structured IBAN/BIC → fewer errors, lower returns
```
